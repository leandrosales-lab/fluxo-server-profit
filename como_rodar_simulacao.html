📋 Guia de Setup

Rodando o Robô em Modo Simulação

Passo a passo completo: do zero ao primeiro sinal no Profit Pro, sem arriscar dinheiro real.

✅
Recomendado para começar. Neste modo, o servidor Python fica rodando e você alimenta os dados do T&T e Book manualmente via chamadas HTTP (ou pelo dashboard web). Perfeito para entender como o sistema funciona antes de conectar ao Profit de verdade.
1
Extraia e abra a pasta do projeto
Descompacte o fluxo_server_v2.zip numa pasta de fácil acesso, por exemplo:
Windows Explorer
# Sugestão de pasta
C:\FluxoServer\
2
Abra o PowerShell na pasta do projeto
No Windows Explorer, segure Shift + clique direito dentro da pasta → "Abrir janela do PowerShell aqui"
PowerShell
cd C:\FluxoServer\fluxo_server
3
Instale as dependências (apenas na primeira vez)
PowerShell
python -m pip install -r requirements.txt
⚠️
Se der erro de "python não encontrado", baixe o Python 3.10+ em python.org e marque a opção "Add to PATH" na instalação.
4
Inicie o servidor em modo manual (simulação)
Escolha o ativo que quer simular. O servidor fica ouvindo na porta 5000:
PowerShell — PETR4
python main.py --asset PETR4 --modo manual
PowerShell — VALE3
python main.py --asset VALE3 --modo manual --min-confidence 0.65
🟢
Se tudo certo, você verá no terminal:
[INFO] FlowServer iniciado em http://127.0.0.1:5000
5
Abra o Dashboard no navegador
Com o servidor rodando, abra o Chrome/Edge e acesse:
Navegador
http://127.0.0.1:5000/dashboard
Você verá o painel de monitoramento em tempo real: sinal atual, fluxo, icebergs detectados, estado do motor.
Endpoints disponíveis
GET
/signal
Sinal atual: BUY / SELL / WAIT
GET
/state
Estado completo do motor
POST
/tick
Injetar negociação do T&T manualmente
POST
/book
Injetar snapshot do Book manualmente
POST
/reset
Resetar motor (início de pregão)
6
Simule negociações enviando ticks via POST
Abra outro PowerShell e envie ticks como se fossem do T&T do Profit. Simule um iceberg da Goldman vendendo em PETR4:
PowerShell — Simular Iceberg Vendedor
# Simula vários players COMPRANDO do iceberg vendedor (Goldman)
# Cada bloco = uma negociação no T&T

# Bradesco comprando 2.100 papéis a R$38,40
Invoke-WebRequest -Uri http://127.0.0.1:5000/tick -Method POST `
  -ContentType "application/json" `
  -Body '{"price":38.40,"qty":2100,"broker":"BRADESCO","side":"C"}'

# Goldman comprando 3.000 — o mesmo iceberg vendedor girou
Invoke-WebRequest -Uri http://127.0.0.1:5000/tick -Method POST `
  -ContentType "application/json" `
  -Body '{"price":38.40,"qty":3000,"broker":"GOLDMAN SACHS","side":"C"}'

# UBS comprando 4.800
Invoke-WebRequest -Uri http://127.0.0.1:5000/tick -Method POST `
  -ContentType "application/json" `
  -Body '{"price":38.40,"qty":4800,"broker":"UBS","side":"C"}'

# JP Morgan comprando 5.200
Invoke-WebRequest -Uri http://127.0.0.1:5000/tick -Method POST `
  -ContentType "application/json" `
  -Body '{"price":38.40,"qty":5200,"broker":"JP MORGAN","side":"C"}'

# Agora consulta o sinal gerado pelo robô
Invoke-WebRequest -Uri http://127.0.0.1:5000/signal | ConvertFrom-Json
💡
O campo side aceita: "C" = compra, "V" = venda. O broker deve ser exatamente como está na lista de corretoras relevantes.
Resposta esperada do /signal após vários ticks:
Resposta JSON — GET /signal
▲ BUY
asset"PETR4"
signal"BUY"
price38.40
stop38.37
confidence0.78
iceberg_renewals4
dominant_broker"GOLDMAN SACHS"
situation"S1_acumulo_RT_venda_romper"
7
Configure o robô NTSL no Profit em modo simulação
No Profit Pro, abra o gráfico do ativo e carregue o robô. Antes de ativar, configure:
Parâmetros do Robô NTSL — Simulação
// Parâmetros que aparecem na tela de configuração do robô:

iQtdePorTrade     = 100       // quantidade de papéis por trade
iHorarioInicio    = 1000      // início às 10h00
iHorarioFimEntrada= 1630      // sem novas entradas após 16h30
fMetaDiaria       = 500.00    // meta de lucro do dia (R$)
fLimitePerdaDiaria= 300.00    // limite de perda do dia (R$)
fConfMinima       = 0.70      // só entra com 70%+ de confiança
iSinalExterno     = 0         // 0=aguarda sinal (alimentado via célula)
🔶
Modo Simulação no Profit: Na barra de ferramentas do robô, certifique-se de que o botão está em "Simulação" (não "Real"). O Profit mostrará as ordens virtualmente sem enviar para a B3.
8
Integre o sinal do servidor com o robô NTSL
O robô NTSL precisa receber o sinal do servidor Python. A forma mais simples é via arquivo de texto que o servidor grava a cada tick:
Arquivo gravado automaticamente: sinal_atual.txt
# Formato do arquivo (atualizado a cada sinal novo):
BUY;38.40;38.37;GOLDMAN SACHS;0.78
# SINAL ; PRECO ; STOP ; CORRETORA ; CONFIANCA
📌
Por enquanto, a integração mais prática para simulação é: você mesmo lê o dashboard e executa as ordens no Profit manualmente em modo simulação. Isso valida a lógica antes de automatizar tudo.
9
Workflow de simulação completo (dia de pregão)
Rotina diária de teste enquanto ainda em simulação:
  • Antes do pregão: python main.py --asset PETR4 --modo manual
  • Abrir dashboard: http://127.0.0.1:5000/dashboard
  • Abrir Profit Pro → ativar robô em modo Simulação
  • Observar o T&T e Book do Profit
  • Quando ver um iceberg, enviar os ticks via PowerShell (ou scripts)
  • Checar o sinal em /signal
  • Executar manualmente no Profit em simulação
  • Ao final do dia: POST /reset para limpar o estado
  • Anotar os resultados e comparar com os sinais gerados

Modo automático. O servidor captura a tela do Profit Pro com OCR e alimenta o motor sozinho. Você não precisa enviar ticks manualmente. Requer calibração da região da tela onde fica o T&T e o Book.
1
Instale o Tesseract OCR
Baixe em: github.com/UB-Mannheim/tesseract/wiki → versão Windows 64-bit
Instalação padrão — deixar em:
C:\Program Files\Tesseract-OCR\tesseract.exe
2
Instale dependências OCR
PowerShell
python -m pip install mss pillow pytesseract opencv-python
3
Rode o modo calibração para mapear a tela
Deixe o Profit aberto com o T&T e Book visíveis, depois rode:
PowerShell
python main.py --asset PETR4 --modo ocr --calibrate
Isso salva screenshots da tela atual para você identificar as coordenadas do T&T e Book.
4
Rode com as regiões mapeadas
Use as coordenadas descobertas na calibração. Exemplo para monitor Full HD (1920×1080):
PowerShell — com regiões
python main.py --asset PETR4 --modo ocr `
  --tt-region   1300,200,580,400 `
  --book-region 0,200,400,500
# formato: left,top,width,height em pixels
⚠️
As coordenadas variam conforme seu monitor e layout do Profit. Ajuste após a calibração.
🔗
Modo avançado. O Profit Pro expõe dados via DDE (Dynamic Data Exchange). O servidor Python lê esses dados diretamente. Mais estável que OCR mas requer configuração do DDE no Windows.
1
Instale pywin32
PowerShell
python -m pip install pywin32
2
No Profit Pro — ative o DDE
Menu Ferramentas → Configurações → DDE. Marque "Habilitar servidor DDE". O servidor DDE do Profit usa o tópico ADVFN.
3
Rode o servidor em modo DDE
PowerShell
python main.py --asset PETR4 --modo dde
❗
O modo DDE só funciona no Windows com o Profit Pro aberto e DDE ativado. Para simular sem o Profit, use o modo Manual.
Todos os parâmetros do servidor
Referência completa — python main.py --help
--asset           PETR4   # ticker do ativo (PETR4, VALE3, ITUB4...)
--modo            manual  # ocr | dde | manual
--host            127.0.0.1 # IP do servidor HTTP
--port            5000    # porta (padrão 5000)
--flow-window     30.0    # janela de análise de fluxo em segundos
--flow-pct        65.0    # % mínimo de fluxo dominante para sinal
--iceberg-min     3       # renovações mínimas para confirmar iceberg
--tick-size       0.01    # tamanho do tick do ativo
--min-confidence  0.60    # confiança mínima para gerar sinal (0.0–1.0)
--calibrate               # (modo ocr) salva screenshots e sai
--tt-region       x,y,w,h # (modo ocr) região do T&T na tela
--book-region     x,y,w,h # (modo ocr) região do Book na tela
Problemas comuns e soluções
❌
Erro: "No module named 'core'"
Você está rodando o python de dentro da pasta errada. Execute dentro de fluxo_server/, não da pasta raiz do zip.
❌
Erro: "Address already in use" (porta 5000)
Já tem um servidor rodando. Mate o processo ou use outra porta: --port 5001
❌
Sinal sempre WAIT / confiança baixa
Normal no início — o motor precisa de dados suficientes. Envie pelo menos 6–8 ticks no mesmo preço com corretoras relevantes (Goldman, Bradesco, Itaú) para ver o sinal BUY/SELL aparecer.
⚠️
Logs do servidor
Os logs ficam em fluxo_server/logs/fluxo_YYYYMMDD.log. Consulte ali se algo não estiver funcionando.
